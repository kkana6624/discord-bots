# プロダクト概要: Discord AND条件ロール自動管理Bot

## 1. プロダクトの目的と解決する課題（Why）

Discordの標準的な権限モデルは「加算方式（OR条件）」で設計されており、「ロールAを持っている、**または**ロールBを持っている」場合に権限を許可することは可能ですが、「ロールAとロールBを**両方持っている場合のみ**」という「論理積（AND条件）」によるアクセス制御を標準機能だけで実現することは不可能です。

本プロダクトは、このDiscord仕様上の制約を解消するため、「特定のロール群（AND条件）が揃ったユーザーに対して、アクセス権限となる第3のロールを自動付与・剥奪する」ことに特化した単機能マイクロサービスです。

## 2. ユースケース（具体的な利用シーン）

* **段階的な権限解放**: 「コミュニティ所属ロール」と「イベント参加申し込み完了ロール」の両方を取得して初めて、「イベント専用チャンネル」の閲覧ロールが付与される。
* **セキュリティ担保**: 対象チャンネルへのアクセス権限を手動で付与するのではなく、Botに自動管理させることで、条件ロールが外れた際の「権限の消し忘れ（フェイルオープン）」をシステム的に防止する。

## 3. コア機能とユーザー体験（What）

本Botは、ユーザーからのコマンド入力等を一切必要としない「インビジブル（不可視）なバックグラウンドワーカー」として振る舞います。

* **リアルタイムな自動付与（Provisioning）**:
サーバー内でユーザーのロールが変更されたことを即座に検知し、指定された2つの「条件ロール（AとB）」が揃った瞬間に、アクセス用の「ターゲットロール（C）」を自動で付与します。
* **リアルタイムな自動剥奪（Deprovisioning）**:
ユーザーから条件ロールA、またはBのいずれかが外れた瞬間に、ターゲットロール（C）を即座に剥奪します。
* **状態の自動修復（Self-Healing / Reconciliation）**:
Discord APIの一時的な障害やBotのコンテナ再起動中に生じた「イベントの取りこぼし」を検知するため、定期的に全メンバーの状態をスキャンし、条件を満たしていないのにロールCを持っているユーザーから強制的に権限を剥奪します。

## 4. プロダクトの制約と前提条件（Constraints）

* **単一サーバー特化**: 汎用的な公開Botではなく、環境変数で指定された単一のDiscordサーバー（Guild）でのみ動作します。
* **完全ステートレス**: Bot自身はデータベース等の状態保持機構を一切持ちません。常に「Discordサーバー上の現在のロール付与状況」を信頼できる唯一の情報源（Single Source of Truth）として扱います。
